Testing a Workflow on a system

Let's walk through a simple example of a test using a Given When Then flow style interpreter.

First, we need to import the package containing the fixture code for our banking application.

import
info.novatec.testit.livingdoc.interpreter.flow.workflow

We'll use the WorkflowInterpreter

Workflow Testing the Workflow interpreter

we arrange the scnenario context using Given keyword.
The method behind it should return a boolean.

Given a method returning true with one parameter parameter

We define the action using the When keyword.
The method behind it should return a boolean.

When a method returning true

We verify the result using the check or check that keyword.
The method behind it should return a value written in the last cell of the row that would be asserted by LivingDoc.

check that the method returns a value of 12345
check the method returns a value of 12345

We could also verify result using the Then or And keywords.
The method behind it should return a boolean.

Then a method returning true
And a method returning true

To check a negative test (returning false) use the keyword And not that

And not that a method returning false

To embed another interpreter type define a method returning a fixtire of that interpreter And not that
